# Deploy NGINX Application with SSL (Self-Signed) in Kubernetes

## Purpose

This document describes how to deploy a simple **NGINX application** on Kubernetes and expose it over **HTTPS** using a **self-signed SSL certificate** via **NGINX Ingress Controller**.

> ⚠️ This approach is intended **only for internal / controlled environments**. Browsers will show **no warning only after the certificate is manually trusted** on client machines.

---

## Architecture Overview

```
Browser (trusted cert)
   ↓ HTTPS
NGINX Ingress Controller (TLS termination)
   ↓ HTTP
NGINX Application Pod
```

---

## Prerequisites

* Kubernetes cluster
* NGINX Ingress Controller installed
* kubectl access
* OpenSSL installed
* DNS hostname pointing to Ingress LoadBalancer IP

Verify ingress controller:

```bash
kubectl get pods -n ingress-nginx
```

---

## Step 1: Generate Self-Signed Certificate (with SAN)

> SAN is mandatory for modern browsers

```bash
openssl req -x509 -nodes -days 365 \
  -newkey rsa:2048 \
  -keyout nginx-app.key \
  -out nginx-app.crt \
  -subj "/CN=nginx.mydomain.internal" \
  -addext "subjectAltName=DNS:nginx.mydomain.internal"
```

Files created:

* `nginx-app.key`
* `nginx-app.crt`

---

## Step 2: Create Kubernetes TLS Secret

```bash
kubectl create secret tls nginx-app-tls \
  --cert=nginx-app.crt \
  --key=nginx-app.key \
  --namespace default
```

Verify:

```bash
kubectl get secret nginx-app-tls
```

---

## Step 3: Deploy NGINX Application

### Deployment YAML

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-app
  labels:
    app: nginx-app
spec:
  replicas: 1
  selector:
    matchLabels:
      app: nginx-app
  template:
    metadata:
      labels:
        app: nginx-app
    spec:
      containers:
      - name: nginx
        image: nginx:latest
        ports:
        - containerPort: 80
```

Apply:

```bash
kubectl apply -f nginx-deployment.yaml
```

---

## Step 4: Create Service

```yaml
apiVersion: v1
kind: Service
metadata:
  name: nginx-app-svc
spec:
  selector:
    app: nginx-app
  ports:
  - port: 80
    targetPort: 80
```

Apply:

```bash
kubectl apply -f nginx-service.yaml
```

---

## Step 5: Create HTTPS Ingress (NGINX)

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: nginx-app-ingress
  annotations:
    nginx.ingress.kubernetes.io/ssl-redirect: "true"
spec:
  ingressClassName: nginx
  tls:
  - hosts:
    - nginx.mydomain.internal
    secretName: nginx-app-tls
  rules:
  - host: nginx.mydomain.internal
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: nginx-app-svc
            port:
              number: 80
```

Apply:

```bash
kubectl apply -f nginx-ingress.yaml
```

---

## Step 6: DNS Configuration

Get Ingress external IP:

```bash
kubectl get ingress nginx-app-ingress
```

Create DNS record:

```
nginx.mydomain.internal → <INGRESS_EXTERNAL_IP>
```

---

## Step 7: Trust the Certificate (Client Side)

### Windows

* Open `nginx-app.crt`
* Install → **Trusted Root Certification Authorities**

### macOS

* Keychain Access → System
* Import cert → set **Always Trust**

### Linux

```bash
sudo cp nginx-app.crt /usr/local/share/ca-certificates/
sudo update-ca-certificates
```

### Firefox

* Settings → Certificates → Authorities → Import

---

## Step 8: Validation

Open browser:

```
https://nginx.mydomain.internal
```

Expected result:

* HTTPS lock icon
* No browser warning
* NGINX welcome page

## Troubleshooting

| Issue           | Cause             | Fix                      |
| --------------- | ----------------- | ------------------------ |
| Browser warning | Cert not trusted  | Import cert properly     |
| Browser warning | SAN missing       | Regenerate cert with SAN |
| TLS ignored     | Using ALB ingress | Use NGINX ingress        |
| 502 error       | Service mismatch  | Verify service name      |
