# 03.1 – Input Variables

This version applies stronger naming conventions and removes remaining hardcoded values.

Updates Made:

* Resource names use consistent project prefix
* Security group name is now a variable
* CIDR variable renamed for clarity
* Logical naming improved

---

provider "aws" {

---

# 2️⃣ variables.tf

All configurable values defined with clean naming.

```hcl
variable "aws_region" {
  description = "AWS region where resources will be created"
  type        = string
  default     = "us-east-1"
}

variable "ami_id" {
  description = "AMI ID for the EC2 instance"
  type        = string
}

variable "instance_type" {
  description = "EC2 instance type"
  type        = string
  default     = "t3.micro"
}

variable "app_server_name" {
  description = "Name tag for the application server"
  type        = string
  default     = "my-app-server"
}

variable "app_security_group_name" {
  description = "Name of the application security group"
  type        = string
  default     = "my-app-sg"
}

variable "allowed_ssh_cidr_blocks" {
  description = "List of CIDR blocks allowed to access SSH"
  type        = list(string)
  default     = ["0.0.0.0/0"]
}
```

Naming Principles Used:

* Lowercase
* snake_case
* Project-prefixed naming
* Clear purpose in variable names

---

# 3️⃣ security_group.tf

Security group now uses improved naming and dynamic name value.

```hcl
resource "aws_security_group" "my_app_sg" {
  name        = var.app_security_group_name
  description = "Allow SSH access to application server"

  ingress {
    from_port   = 22
    to_port     = 22
    protocol    = "tcp"
    cidr_blocks = var.allowed_ssh_cidr_blocks
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }

  tags = {
    Name = var.app_security_group_name
  }
}
```

Improvements:

* Logical resource name: my_app_sg
* Security group name is variable-driven
* CIDR variable renamed to allowed_ssh_cidr_blocks

---

# 4️⃣ ec2.tf

Resource renamed with consistent application prefix.

```hcl
resource "aws_instance" "my_app_server" {
  ami           = var.ami_id
  instance_type = var.instance_type

  vpc_security_group_ids = [aws_security_group.my_app_sg.id]

  tags = {
    Name = var.app_server_name
  }
}
```

Improvements:

* Resource logical name: my_app_server
* Security group reference updated
* Name tag is fully dynamic

# 5️⃣ Example Apply Command

Since ami_id has no default, pass it explicitly.

```
terraform apply \
  -var="ami_id=ami-0f3caa1cf4417e51b" \
  -var="instance_type=t3.small" \
  -var='allowed_ssh_cidr_blocks=["10.0.0.0/24"]'
```
