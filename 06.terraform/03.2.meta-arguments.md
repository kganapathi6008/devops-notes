# EC2 Project – Advanced Meta Arguments Example (Dev Environment)

This document extends your current ec2-project by adding:

* for_each
* count (indirectly via computed expansion)
* depends_on
* lifecycle
* dynamic blocks
* proper map-based configuration

Your requirement:

* Multiple services (app1, app2)
* Each service defines its own:

  * ami_id
  * instance_type
  * instance_count
  * enable_monitoring
* One shared dev security group
* Security group rules defined using map
* Region and environment common for all resources

# variables.tf

```hcl
variable "aws_region" {
  type = string
}

variable "environment" {
  type = string
}

variable "org_name" {
  type        = string
  description = "Organization name used in resource naming"
}

variable "ec2_instances" {
  description = "Map of EC2 service definitions"
  type = map(object({
    ami_id            = string
    instance_type     = string
    instance_count    = number
    enable_monitoring = bool
  }))
}

variable "security_group_rules" {
  description = "Map of security group ingress rules"
  type = map(object({
    port        = number
    description = string
    cidr_blocks = list(string)
  }))
}
```

# main.tf

## 1. Security Group (Using dynamic block)

```hcl
resource "aws_security_group" "ec2_security_group" {
  name        = "${var.org_name}-${var.environment}-ec2-sg"
  description = "Security group for ${var.environment} EC2 instances"

  dynamic "ingress" {
    for_each = var.security_group_rules

    content {
      from_port   = ingress.value.port
      to_port     = ingress.value.port
      protocol    = "tcp"
      description = ingress.value.description
      cidr_blocks = ingress.value.cidr_blocks
    }
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }

  lifecycle {
    create_before_destroy = true
  }
}
```

Meta arguments used:

* dynamic
* for_each
* lifecycle

---

## 2. Expand EC2 Instances Based on instance_count

We convert the map into a flat structure.

```hcl
locals {
  computed_ec2_instances = flatten([
    for service_name, config in var.ec2_instances : [
      for i in range(config.instance_count) : {
        instance_key        = "${service_name}-${i}"
        service_name      = service_name
        index             = i
        ami_id            = config.ami_id
        instance_type     = config.instance_type
        enable_monitoring = config.enable_monitoring
      }
    ]
  ])
}
```

This simulates count per service using for_each.

---

## 3. EC2 Instances (Using for_each + depends_on + lifecycle)

```hcl
resource "aws_instance" "example" {
  for_each = {
    for instance in local.computed_ec2_instances :
    instance.instance_key => instance
  }

  ami                    = each.value.ami_id
  instance_type          = each.value.instance_type
  monitoring             = each.value.enable_monitoring
  vpc_security_group_ids = [aws_security_group.ec2_security_group.id]

  tags = {
    Name        = "${var.org_name}-${each.value.service_name}-${var.environment}-${each.value.index}"
    Environment = var.environment
    Service     = each.value.service_name
  }

  depends_on = [aws_security_group.ec2_security_group]

  lifecycle {
    ignore_changes = [tags]
  }
}

```

Meta arguments used:

* for_each
* depends_on
* lifecycle

# What Meta Arguments We Used

for_each

* Creates instances per expanded map

count (via range expansion)

* Simulates instance_count per service

lifecycle

* create_before_destroy (SG)
* ignore_changes (EC2 tags)

depends_on

* Ensures SG is created first

dynamic block

* Generates multiple ingress rules

locals

* Converts structured input into expanded instance list
