# PART 5 – META ARGUMENTS

Meta arguments control how resources behave.

---

## 1. count

Creates multiple identical resources.

```hcl
resource "aws_instance" "app_server" {
  count         = 2
  ami           = "ami-123456"
  instance_type = "t3.micro"
}
```

Access individual instance:

```
aws_instance.app_server[0]
aws_instance.app_server[1]
```

Use when:

* Resources are identical

Limitation:

* Index-based (can cause recreation issues)

---

## 2. for_each

Creates multiple resources using map or set.

Example using map:

```hcl
variable "instances" {
  default = {
    app1 = "t3.micro"
    app2 = "t3.small"
  }
}

resource "aws_instance" "app_server" {
  for_each      = var.instances
  ami           = "ami-123456"
  instance_type = each.value

  tags = {
    Name = each.key
  }
}
```

Better than count because:

* Uses keys
* No index shifting problem

---

## 3. depends_on

Explicit dependency.

```hcl
resource "aws_instance" "app" {
  ami           = "ami-123456"
  instance_type = "t3.micro"

  depends_on = [aws_security_group.app_sg]
}
```

Use only when implicit dependency is not detected.

---

## 4. lifecycle

Controls creation and destruction behavior.

### create_before_destroy

```hcl
lifecycle {
  create_before_destroy = true
}
```

Creates new resource before destroying old one.

---

### prevent_destroy

```hcl
lifecycle {
  prevent_destroy = true
}
```

Prevents accidental deletion.

---

### ignore_changes

```hcl
lifecycle {
  ignore_changes = [tags]
}
```

Terraform ignores changes to tags.

---

# PART 6 – DYNAMIC BLOCKS

Used when nested blocks must be generated dynamically.

Example: Multiple ingress rules

```hcl
variable "ports" {
  default = [22, 80, 443]
}

resource "aws_security_group" "app_sg" {
  name = "app-sg"

  dynamic "ingress" {
    for_each = var.ports
    content {
      from_port   = ingress.value
      to_port     = ingress.value
      protocol    = "tcp"
      cidr_blocks = ["0.0.0.0/0"]
    }
  }
}
```

Without dynamic block, you would manually write multiple ingress blocks.
