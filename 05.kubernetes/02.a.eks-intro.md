# Amazon EKS Cluster Overview and Pre‑Requisites

## **1. What is an Amazon EKS Cluster?**

Amazon Elastic Kubernetes Service (EKS) is a fully managed Kubernetes service provided by AWS. It allows you to run containerized applications using Kubernetes without having to manage the Kubernetes control plane yourself.

### **Key Components**

* **Managed Control Plane**: AWS runs and maintains the Kubernetes control plane. This includes the API server and etcd database, spread across multiple Availability Zones for high availability. You do not need to install, secure, or scale the master nodes.

* **Worker Nodes (Your Responsibility)**: The applications and workloads run on worker nodes that you manage. These can be:

  * Amazon EC2 instances
  * AWS Fargate serverless compute

* **AWS Integrations**: EKS integrates with services like IAM, VPC, CloudWatch, ELB, and Auto Scaling, providing a secure and scalable environment.

### **Benefits of EKS**

* High availability and resilience
* Automatic control plane scaling and patching
* Tight integration with the AWS ecosystem
* Ability to run Kubernetes applications without maintaining the master nodes

## **2. Pre‑Requisites to Install and Access an EKS Cluster**

Before creating or using an EKS cluster, the following requirements must be met.

### **2.1 AWS Account and IAM Permissions**

You need an AWS account with permissions to create and manage:

* EKS clusters
* EC2 instances and networking
* CloudFormation stacks
* IAM roles and policies

For learning environments, the **AdministratorAccess** policy is sufficient.

---

### **2.2 Required CLI Tools**

To create and interact with your EKS cluster, install the following tools:

#### **a) AWS CLI**

Used to authenticate with AWS and manage services.
Configure it using:

```
aws configure
```

#### **b) kubectl**

The Kubernetes command-line tool used to interact with your cluster resources.

#### **c) eksctl**

A simple and recommended CLI tool specifically for creating and managing EKS clusters.

---

### **2.3 Networking Requirements (VPC)**

Your EKS cluster requires a properly configured VPC that includes:

* At least **2 or 3 subnets** in different Availability Zones
* Internet Gateway or NAT Gateway (depending on node visibility)
* Security groups for worker nodes and the control plane
* Proper route tables for public/private subnet communication

`eksctl` can automatically create the required VPC for you.

---

### **2.4 IAM Roles for EKS**

EKS needs IAM roles for cluster operations and node groups, including:

* **EKS Cluster Role**
* **Node Instance Role**
* **Fargate Execution Role** (only for Fargate workloads)

These roles are automatically generated when using `eksctl`.

---

### **2.5 Connecting to the EKS Cluster**

Once the cluster is created, you must update your kubeconfig:

```
aws eks update-kubeconfig --region <region> --name <cluster-name>
```

Verify access:

```
kubectl get nodes
kubectl get pods -A
```

## **Summary**

| Area          | Managed By   | Description                                            |
| ------------- | ------------ | ------------------------------------------------------ |
| Control Plane | AWS          | Automatically scaled, patched, and highly available    |
| Worker Nodes  | You          | EC2 or Fargate instances running application workloads |
| Networking    | You          | VPC, subnets, routing, and security groups             |
| IAM Roles     | You          | Required for cluster and node operations               |
| CLI Tools     | You          | AWS CLI, kubectl, eksctl for cluster management        |

This document provides a clear overview of EKS and the essential prerequisites needed to install and access an EKS cluster.
